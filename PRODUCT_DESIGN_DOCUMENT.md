# AndroidFile 产品设计文档 (PRD)

## 1. 产品概述
### 1.1 背景与目标
macOS 系统原生不支持安卓设备的 MTP (Media Transfer Protocol) 协议，导致用户在 Mac 上管理安卓手机文件时体验不佳。
**AndroidFile** 旨在打造一款原生、轻量、高效的 macOS 安卓文件管理器，提供类似 Finder 的操作体验，并整合开发者常用的工具。

### 1.2 核心价值
*   **原生性能**: 使用 Swift/SwiftUI 构建，低内存占用，高响应速度。
*   **双模连接**: 同时兼容普通用户的 MTP 模式和开发者的 ADB 模式。
*   **工具集成**: 内置图片/视频预览和 `scrcpy` 投屏功能，减少工具链切换。

---

## 2. 用户分析
*   **普通安卓用户**: 需要在 Mac 和手机之间互传照片、视频、下载文件。
*   **安卓开发者**: 需要频繁访问 `/sdcard` 目录，检查应用日志、截图，进行视频演示及屏幕调试。

---

## 3. 功能需求 (Functional Requirements)

### 3.1 设备管理
*   **自动发现**: 实时监听 USB 连接，自动识别并列出已连接的安卓设备。
*   **多协议识别**: 自动识别设备是通过 ADB (调试模式) 还是 MTP (文件传输模式) 连接。
*   **信息展示**: 在界面显著位置展示设备型号、序列号及连接协议。

### 3.2 文件系统操作
*   **目录导航**: 支持层级目录浏览，支持面包屑路径导航及“返回上一级”操作。
*   **快捷跳转**: 侧边栏提供“相册 (DCIM)”、“下载 (Download)”、“内部存储 (Root)”的快捷访问入口。
*   **兼容性**: 深度适配安卓软链接（Symlink），确保 `/sdcard` 等特殊路径能正确展示。
*   **文件操作**:
    *   **双击**: 文件夹进入、图片/视频开启预览。
    *   **右键菜单**: 支持“拷贝到 Mac”及“删除 (Delete)”。

### 3.3 多媒体预览系统
*   **动态侧边栏**: 仅在双击多媒体文件时，从右侧滑出预览面板。
*   **图片预览**: 支持 jpg, png, webp, heic 等格式的大图渲染。
*   **视频播放**: 
    *   支持 mp4, mov, mkv 等主流格式。
    *   内置原生播放器控件（播放/暂停、进度调节、音量）。
*   **比例自适应**: 自动解析视频/图片的长宽比，动态调整预览窗口高度，适配横竖屏。

### 3.4 文件传输与上传
*   **下载至本地**: 支持通过右键菜单将文件异步下载至 Mac 的“下载”文件夹。
*   **拖拽上传**: 支持从 Mac 访达直接拖拽文件/文件夹至应用窗口，触发全屏遮罩反馈并自动上传至手机当前目录。
*   **进度管理**: 顶部显示实时传输状态，支持后台异步任务。

### 3.5 扩展工具
*   **屏幕镜像 (Screen Mirroring)**: 集成 `scrcpy`，在 ADB 模式下支持一键开启手机投屏。

---

## 4. UI/UX 设计规范

### 4.1 交互架构
采用 **NavigationSplitView** (两栏动态布局) 架构：
*   **左侧边栏 (Sidebar)**: 品牌标识、设备列表、快捷路径。
*   **右侧详细区 (Detail)**: 
    *   左侧为主文件列表及工具栏。
    *   右侧为动态滑出的多媒体预览面板（仅在选中多媒体时显示）。
    *   拖拽交互：文件移入时显示全屏蓝色高亮遮罩及“拖拽上传”提示。

### 4.2 视觉风格
*   **设计语言**: 遵循 macOS 标准设计规范，使用原生 Squircle 应用图标。
*   **配色方案**: 采用“安卓绿” (#3DDC84) 渐变。
*   **动画效果**: 预览面板使用 Spring 弹簧动画，支持平滑的比例切换和滑入过渡。

---

## 5. 技术架构 (Technical Architecture)

### 5.1 核心技术栈
*   **开发语言**: Swift 5.9+
*   **UI 框架**: SwiftUI + AppKit (AVPlayerView)
*   **底层通信**: ADB (Shell Process) + MTP (libmtp bridge)
*   **最低版本**: macOS 13.0 (Ventura)

### 5.2 模块划分
*   **Core 层**: `FileProvider` 协议化设计，实现逻辑与驱动分离。
*   **ADB 实现**: 封装 `Process` 调用，自研 Robust 解析器处理 `ls` 复杂输出。
*   **预览引擎**: 采用“按需拉取 + 临时缓存 + 异步渲染”策略。

### 5.3 多媒体处理方案
*   **长宽比检测**: 利用 `AVAsset` 异步加载视频轨道，解析 `naturalSize` 配合 `preferredTransform` 计算物理显示比例。
*   **播放器稳定性**: 使用 `NSViewRepresentable` 封装系统原生 `AVPlayerView`，确保大视频播放不闪退。
*   **竞态保护**: 引入 `Task` 取消机制，确保在快速切换预览时旧的传输进程被及时终止。
