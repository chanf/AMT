# AndroidFile 产品设计文档 (PRD)

## 1. 产品概述
### 1.1 背景与目标
macOS 系统原生不支持安卓设备的 MTP (Media Transfer Protocol) 协议，导致用户在 Mac 上管理安卓手机文件时体验不佳。
**AndroidFile** 旨在打造一款原生、轻量、高效的 macOS 安卓文件管理器，提供类似 Finder 的操作体验，并整合开发者常用的工具。

### 1.2 核心价值
*   **原生性能**: 使用 Swift/SwiftUI 构建，低内存占用，高响应速度。
*   **双模连接**: 同时兼容普通用户的 MTP 模式和开发者的 ADB 模式。
*   **工具集成**: 内置图片预览和 `scrcpy` 投屏功能，减少工具链切换。

---

## 2. 用户分析
*   **普通安卓用户**: 需要在 Mac 和手机之间互传照片、视频、下载文件。
*   **安卓开发者**: 需要频繁访问 `/sdcard` 目录，检查应用日志、截图，并进行屏幕调试。

---

## 3. 功能需求 (Functional Requirements)

### 3.1 设备管理
*   **自动发现**: 实时监听 USB 连接，自动识别并列出已连接的安卓设备。
*   **多协议识别**: 自动识别设备是通过 ADB (调试模式) 还是 MTP (文件传输模式) 连接。
*   **信息展示**: 在界面显著位置展示设备型号、序列号及连接协议。

### 3.2 文件系统操作
*   **目录导航**: 支持层级目录浏览，支持面包屑路径导航及“返回上一级”操作。
*   **快捷跳转**: 侧边栏提供“相册 (DCIM)”、“下载 (Download)”、“内部存储 (Root)”的快捷访问入口。
*   **文件列表**: 展示文件名、大小、类型等基本属性。
*   **双击交互**: 
    *   双击文件夹：进入目录。
    *   双击图片：开启预览模式。

### 3.3 图片预览系统
*   **动态侧边栏**: 仅在双击图片文件时，从右侧滑出预览面板。
*   **实时拉取**: 针对 ADB 设备，通过缓存机制从手机实时下载图片并渲染大图。
*   **元数据展示**: 在预览区展示文件路径、精确字节数及修改日期。

### 3.4 文件传输与管理
*   **下载至本地**: 右键点击文件，支持将其拷贝至 Mac 的“下载”文件夹。
*   **任务管理**: 顶部显示实时传输进度条，支持异步后台传输。

### 3.5 扩展工具
*   **屏幕镜像 (Screen Mirroring)**: 集成 `scrcpy`，在 ADB 模式下支持一键开启手机投屏。

---

## 4. UI/UX 设计规范

### 4.1 交互架构
采用 **NavigationSplitView** (两栏动态布局) 架构：
*   **左侧边栏 (Sidebar)**: 品牌标识、设备列表、快捷路径。
*   **右侧详细区 (Detail)**: 
    *   左侧为主文件列表及工具栏。
    *   右侧为动态弹出的预览面板（仅在选中图片时显示）。

### 4.2 视觉风格
*   **设计语言**: 遵循 macOS Big Sur 及更高版本的设计规范（圆角、毛玻璃效果、SF Symbols）。
*   **配色方案**: 采用安卓标志性的“安卓绿” (#3DDC84) 作为品牌色和图标主色。
*   **动画效果**: 预览面板使用 Spring 弹簧动画，支持平滑的滑入/滑出过渡。

---

## 5. 技术架构 (Technical Architecture)

### 5.1 核心技术栈
*   **开发语言**: Swift 5.9+
*   **UI 框架**: SwiftUI
*   **构建工具**: Swift Package Manager (SPM)
*   **最低版本支持**: macOS 13.0 (Ventura)

### 5.2 模块划分
*   **Core 层**: 定义 `FileProvider` 抽象协议，解耦 UI 与底层协议。
*   **ADB 实现**: 通过 Swift `Process` 调用 `adb` 命令行工具，解析 Shell 输出。
*   **MTP 实现**: 预留 `libmtp` C 库桥接接口。
*   **UI 层**: 响应式组件化开发，确保状态驱动界面。

### 5.3 图片预览核心技术方案
*   **方案设计思路**: 采用“按需拉取 + 临时缓存 + 异步渲染”策略。
*   **异步并发模型**: 整个拉取过程封装在 Swift Concurrency (`Task`) 中，避免阻塞 UI 主线程。
*   **临时存储管理**: 预览图存储在系统临时目录，文件名全局唯一，随系统清理机制自动回收。
*   **性能优化**: 采用“懒加载”策略，仅在用户明确交互时产生 USB 数据传输。
