# AndroidFile 产品设计文档 (PRD)

## 1. 产品概述
### 1.1 背景与目标
macOS 系统原生不支持安卓设备的 MTP (Media Transfer Protocol) 协议，导致用户在 Mac 上管理安卓手机文件时体验不佳。
**AndroidFile** 旨在打造一款原生、轻量、高效的 macOS 安卓文件管理器，提供类似 Finder 的操作体验，并整合开发者及高级用户常用的全套工具。

### 1.2 核心价值
*   **原生性能**: 使用 Swift/SwiftUI 构建，极速响应，低资源占用。
*   **双模连接**: 完美兼容 ADB (调试模式) 和 MTP (文件传输模式)。
*   **全能预览**: 无需导出，直接在应用内预览图片、视频、音频、文本及 APK 详情。
*   **高效管理**: 集成批量操作、拖拽上传、应用一键安装卸载。

---

## 2. 用户分析
*   **普通安卓用户**: 需要简单直观地互传多媒体文件。
*   **安卓开发者/极客**: 需要深入访问 `/sdcard`，安装/卸载 APK，查看日志，进行屏幕镜像调试。

---

## 3. 功能需求 (Functional Requirements)

### 3.1 设备管理
*   **自动发现**: 实时监听 USB 插拔，自动列出已连接设备。
*   **信息展示**: 显著展示设备型号、序列号及连接协议（ADB/MTP）。

### 3.2 文件系统操作 (Desktop-Class)
*   **目录导航**: 支持面包屑导航、返回上一级、路径展示。
*   **快捷直达**: 侧边栏提供“相册”、“下载”、“内部存储”快捷入口。
*   **高级多选**: 
    *   **点选**: 单击选中。
    *   **加选**: Command + 点击。
    *   **连选**: Shift + 点击实现范围选择。
*   **批量操作**: 支持对多个选中项进行批量“下载到 Mac”或“物理删除”。
*   **软链接兼容**: 深度适配安卓 Symlink，确保 `/sdcard` 等特殊路径准确展示。

### 3.3 全格式预览系统 (Preview Suite)
*   **图片**: 实时渲染大图，支持常见格式。
*   **视频**: 内置 AVPlayerView，支持播放控制，**窗口高度比例自适应**。
*   **音频**: 支持 mp3, m4a 等格式播放，具备专属音乐播放 UI。
*   **文本**: 支持代码/文本（json, txt, log 等）预览，具备 **512KB 安全读取机制**及等宽字体渲染。
*   **APK 详情**: 展示应用图标、名称、版本号及版本代码。

### 3.4 应用管理 (APK/App Management)
*   **一键安装**: 双击或右键 APK 文件即可通过 `/data/local/tmp` 绕过权限拦截进行安装。
*   **应用列表**: 独立模块展示所有第三方应用。
*   **名称解析**: 集成 **AAPT 引擎**，将晦涩的包名解析为真实桌面显示名称。
*   **极速性能**: 引入 **AppCacheManager** 持久化缓存及 **TaskGroup 并发加载**，实现列表秒开。
*   **一键卸载**: 在详情页提供显式的应用卸载功能。

### 3.5 传输与同步
*   **双向传输**: 右键“下载到 Mac”；从访达直接“拖拽上传”到手机当前目录。
*   **进度反馈**: 顶部状态栏及列表行内同步展示传输进度。

### 3.6 调试工具
*   **屏幕镜像**: ADB 模式下集成 `scrcpy` 投屏按钮。

---

## 4. UI/UX 设计规范

### 4.1 交互架构
采用 **NavigationSplitView** 的三栏动态架构：
*   **Sidebar**: 品牌、设备列表、快捷入口、应用列表切换。
*   **Content**: 文件/应用列表主体，支持 macOS 原生风格的**浅蓝色选中高亮**。
*   **Detail**: 动态弹出的多媒体预览或应用详情面板。

### 4.2 视觉与反馈
*   **原生风格**: 遵循 macOS Ventura+ 设计准则，采用 Squircle 图标及 SF Symbols。
*   **交互动效**: 预览面板滑入使用 Spring 弹簧动画；拖拽上传时显示全屏高亮遮罩反馈。

---

## 5. 技术架构 (Technical Architecture)

### 5.1 核心技术实现
*   **解析引擎**: 封装 `Process` 调用 ADB/AAPT/Unzip 命令行工具。
*   **多媒体处理**: 利用 `AVAsset` 提取元数据（用于比例自适应），使用 `NSViewRepresentable` 封装原生 `AVPlayerView` 保证播放稳定性。
*   **缓存系统**: `JSON` 格式本地持久化，管理包名与应用名称的映射。
*   **并发控制**: 使用 Swift Concurrency 管理后台任务，通过 `loadingTask.cancel()` 防止竞态冲突。
