# AndroidFile 产品设计文档 (PRD)

## 1. 产品概述
### 1.1 背景与目标
macOS 系统原生不支持安卓设备的 MTP (Media Transfer Protocol) 协议，导致用户在 Mac 上管理安卓手机文件时体验不佳。
**AndroidFile** 旨在打造一款原生、轻量、高效的 macOS 安卓助理工具，提供类似 Finder 的操作体验，并整合开发者及高级用户常用的全套监控与管理工具。

### 1.2 核心价值
*   **原生性能**: 使用 Swift/SwiftUI 构建，极速响应，低资源占用。
*   **双模连接**: 完美兼容 USB (ADB/MTP) 和无线 Wi-Fi 连接。
*   **全能预览**: 无需导出，直接在应用内预览图片、视频、音频、文本及 APK 详情。
*   **深度监控**: 实时可视化展示设备 CPU、内存、存储及系统温度。

---

## 2. 用户分析
*   **普通安卓用户**: 需要简单直观地互传多媒体文件，查看手机状态。
*   **安卓开发者/极客**: 需要深入访问文件系统，安装/卸载 APK，进行性能调优及屏幕镜像调试。

---

## 3. 功能需求 (Functional Requirements)

### 3.1 设备与连接管理
*   **多模式识别**: 自动区分 USB 与 **WI-FI** 连接，并在侧边栏通过图标显著标识。
*   **远程管理中心**: 
    *   **手动连接**: 支持输入 IP 地址和端口建立 Wi-Fi ADB 连接。
    *   **局域网扫描**: 自动探测网段内开放了 5555 端口的安卓设备，支持一键连接。
*   **无线开关**: 支持 USB 设备一键开启无线调试模式，或关闭无线端口恢复 USB 模式。

### 3.2 桌面级文件系统
*   **多级导航**: 支持面包屑路径、返回上一级、软链接（Symlink）穿透显示。
*   **交互规范**: 
    *   支持 macOS 原生的 **Command 点选** 与 **Shift 连选**（范围选择）。
    *   选中项采用系统级**浅蓝色高亮样式**。
*   **批量操作**: 支持对多个选中项进行批量“下载到 Mac”或“物理删除”。
*   **拖拽上传**: 支持从 Mac 访达直接拖拽至窗口，通过全屏蓝色遮罩提供明确反馈。

### 3.3 全格式预览引擎 (Preview Suite)
*   **图片**: 支持主流格式大图预览。
*   **视频**: 基于 `AVPlayerView` 封装，支持**播放控制**及**窗口比例自适应**（横竖屏适配）。
*   **音频**: 具备专属音乐播放 UI 模式。
*   **文本**: 支持代码及文本预览，内置 **512KB 安全读取机制**及等宽字体渲染。
*   **APK 详情**: 自动提取并展示 **App 真实图标**、应用名、包名及版本信息。

### 3.4 性能监控仪表盘 (Performance Dashboard)
*   **实时折线图**: 高频采集并绘制 **CPU 使用率** 和 **内存占用** 曲线。
*   **温度监控**: 实时追踪**系统核心温度**，支持基于温度区间的颜色预警（绿/橙/红）。
*   **存储可视化**: 采用自定义环形图展示分区容量占用。
*   **电池健康**: 展示电量百分比、充放电状态及电池温度。

### 3.5 应用管理 (App Management)
*   **智能列表**: 集成 **AAPT 引擎**，将包名异步解析为真实中文名称。
*   **极致速度**: 引入 **AppCacheManager** 持久化缓存与 **TaskGroup 并行加载**。
*   **安全安装**: 通过 `/data/local/tmp` 中转安装，解决 Android 13+ 权限拦截问题。

---

## 4. UI/UX 设计规范

### 4.1 布局架构
采用 **NavigationSplitView** 动态响应式布局：
*   **Sidebar**: 品牌标识、设备列表（含状态标识）、管理模块切换（文件/应用/远程/性能）。
*   **Content**: 列表主体，具备手动刷新功能。
*   **Detail**: 仅在双击媒体或应用时滑出的沉浸式预览/详情面板。

### 4.2 动效设计
*   **过渡动画**: 侧边栏滑入采用 Spring 弹簧动效。
*   **反馈引导**: 拖拽上传时的全屏高亮、删除/安装时的顶部状态消息反馈。

---

## 5. 技术架构 (Technical Architecture)

### 5.1 核心技术实现
*   **指令层**: 封装异步 `Process` 调用，实现高效的 ADB 指令管道。
*   **网络层**: 基于 `Network.framework` 实现高性能局域网端口探测。
*   **资源解析**: 结合 `aapt` 元数据扫描与 `unzip` 流式资源提取。
*   **性能保障**: 全链路 Swift Concurrency 任务管理，支持竞态取消及主线程 UI 保证。
